typedef int(* shardcache_client_get_aync_data_cb)(char *node, void *key, size_t klen, void *data, size_t dlen, int error, void *priv)

Callback passed to shardcache_client_get_async() to retrieve the data asynchronously.

Parameters
nodeThe node from which we are receiving the new chunk of data
keyA valid pointer to the key
klenThe length of the key
dataThe pointer to the chunk of data
dlenThe length of the current chunk of data
errorTrue (1) if there was an error. data will point to NULL and dlen will be 0 as well
privThe priv pointer passed to shardcache_client_get_async()
Note
data == NULL and dlen == 0 indicates that the opartion finished error will be set to 1 if the operation failed, 0 otherwise

int shardcache_client_multi_command_max_wait ( shardcache_client_t *  c,
int  new_value 
)

Get and/or set the maximum time (in seconds) to wait on multi commands (get_multi/set_multi) when no data is arriving.

Note
This setting represents a timeout which is reset each time some data is received and it prevents the get_multi()/set_multi() commands from hanging indefinitely waiting for data. A value of 0 indicates no timeout
Parameters
cA valid pointer to a shardcache_client_t structure
new_valueIf greater or equal to 0 the new value will be set. Otherwise the old value will be queried but no new value will be set. A value of 0 indicates no timeout.
Returns
The previously configured max_wait timeout (still valid if no new value has been provided)
int shardcache_client_use_random_node ( shardcache_client_t *  c,
int  new_value 
)

Get and/or set the random_node mode on a shardcache client instance. When on a single (random) node will be used for all the commands instead of using the chash algorithm to determine the owner.

Parameters
cA valid pointer to a shardcache_client_t structure
new_valueIf false, the chash algorithm will be used against the list of known node to determine the owner of a key and commands will be sent only to owners.
If true a single random node will be used instead
